Python爬虫-背景调研

发表于 2016-12-17 | 分类于 Python | | 阅读次数 | 热度 °C

在深入讨论爬取一个网站之前，我们首先需要思考自己爬取的行为是否合法，然后对目标站点的规模和结构进行一定程度的了解。网站自身的robots.txt和Sitemap文件都可以为我们提供一定的帮助，此外还有一些能提供更详细信息的外部工具，比如Google搜索和WHOIS。

网络爬虫是否合法

网络爬虫目前还处于早期的蛮荒阶段，”允许哪些行为”这种基本秩序还处于建设之中。从目前的实践来看，如果抓取数据的行为用于个人所用，则不存在问题；如果数据用于转载，那么抓取的数据类型就非常关键了。

无论如何，当你抓取某个网站的数据时，请记住自己是该网站的访客，应当约束自己的抓取行为，否则他们可能会封禁的你IP，甚至采取更进一步的法律行动。

检查robots.txt

大多数网站都会定义robots.txt文件，这样可以让爬虫了解爬取该网站时存在哪些限制。这些限制虽然仅仅作为建议给出，但是良好的网络公民都应该遵守这些限制。在爬取之前，检查robots.txt文件这一宝贵资源可以最小化爬虫被封禁的可能。而且还能发现和网站结构相关的线索。

估算网站大小

目标网站的大小会影响我们如何进行爬取。

估算一个网站大小的一个简便方法是检查搜索引擎爬虫的结果，因为搜索引擎很可能已经爬取过我们感兴趣的网站。我们可以用site关键词过滤域名结果，从而获取该信息。

比如Google搜索site:example.webscraping.com/viewGoogle估算该网站拥有117个网页，这和实际情况差不多。不过对于更大型的网站，我们发现Google估算并不十分准确。这种附加的过滤条件非常有用，因为在理想情况下，你只希望爬取网站中包含有用数据的部分，而不是爬取网页的每个页面。

识别网站所用技术

构建网站所使用的技术类型也会对我们如何爬取产生影响。有一个十分有用的工具可以检查网站构建的技术类型——builtwith模块。

安装：
pip install builtwith

寻找网站所有者

对于一些网站，我们可能会关心其所有者是谁。比如，我们已知网站的所有者会封禁网络爬虫，那么我们最好把下载速度控制得更加保守一些。为了找到网站的所有者，我们可以使用WHOIS协议查询域名的注册者是谁。Python中有一个针对该协议的封装库，其文档地址为https://pypi.python.org/pypi/python-whois，我们可以通过pip进行安装。
pip install python-whois
